### 8. Why the Generator Failed to Make a Mouse Layer

1. **Missing gene pool:** `MB1`–`MB5` are not in the v2 shortcut pool. The optimizer cannot place what it doesn't know about.
2. **No cursor keys in pool:** Left/Right/Up/Down arrows are not in the pool as individual shortcuts. They appear as combos in canonical but are overwritten.
3. **Dynamic workflow clustering:** The optimizer clusters by co-occurrence in usage data. Mouse button usage is not tracked in `shortcut_usage.jsonl` (the AHK logger only records keyboard shortcuts, not mouse clicks).
4. **No right-hand bias parameter:** The optimizer has no concept of "right-hand 5x more important for mouse tasks." It optimizes for global effort + adjacency, not task-specific hand dominance.
5. **Layer roles are emergent, not assigned:** The optimizer doesn't know "L2 should be mouse." It places whatever shortcuts co-occur. Since mouse clicks aren't in the data, no mouse layer emerges.

---

### 9. Recommendations for Mouse Layer Support

1. **Add mouse buttons to shortcut pool:** Add `MB1`, `MB2`, `MB3`, `MB4`, `MB5` to the v2 optimizer's `shortcut_pool.jsonl`. Mark them as `category: mouse` and `preferred_hand: right`.

2. **Add cursor keys to pool:** Add `Left`, `Right`, `Up`, `Down` as optimizable shortcuts with `preferred_hand: right` and `category: mouse`.

3. **Add scroll behaviors:** Add `ScrollUp`, `ScrollDown` as distinct shortcuts (ZMK supports these as mouse behaviors).

4. **Add right-hand bias penalty:** In the optimizer's fitness function, add a penalty for placing `category: mouse` shortcuts on the left hand. A 5x weight would mean left-hand placement of mouse buttons costs 5x more effort than right-hand placement.

5. **Create a dedicated toggle layer for mouse:** Reserve one layer (e.g., L2 or L5) as a toggle-only mouse layer. Set `layer_mode: toggle` in the config so the optimizer knows this layer should be locked, not held.

6. **Manual ZMK Studio fallback:** Until the optimizer can handle mouse buttons, manually add them to a toggle layer via ZMK Studio. Use the right-hand positions listed in section 7 above.

7. **Track mouse usage:** Extend `shortcut_usage.jsonl` to log mouse button clicks alongside keyboard shortcuts. This gives the optimizer data to cluster mouse actions into a layer.

1. **Check L0 thumbs:** Each thumb should access a layer that makes sense for your most common workflows. If you spend 80% of time in browser + VS Code, the most-used workflow should be on the easiest thumb hold (typically left inner thumb or right inner thumb).
