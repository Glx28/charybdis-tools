## Familiarity Weight — Soft Cross-Layer Consistency Bias

### Concept

When the same shortcut appears on multiple layers, users benefit if it sits at the same physical position every time. Familiarity weight is a **soft bias** (not a hard rule) that rewards duplicate shortcuts for being in identical (x,y) coordinates across layers. The reward scales with importance, so high-value shortcuts get the strongest consistency pressure while rarely used keys can still move for workflow reasons.

### Why a bias, not a rule

A hard rule would freeze every duplicate shortcut and prevent workflow optimization. A bias makes sameness the default but allows the optimizer to break consistency when a different position is genuinely better for a specific layer. The expected outcome: common shortcuts drift toward stable muscle-memory positions, while distinctive workflow-specific shortcuts remain free to cluster.

### Proposed implementation

Add a `FamiliarityFactor` with a small weight (e.g., 0.5–2.0). For each shortcut on k ≥ 2 layers:
- Find the most common (x,y) position.
- Reward each assignment at that position by `importance * 1.0`.
- Lightly penalize deviations by `importance * distance_to_modal * 0.2`.
This keeps the pressure soft: a shortcut can move if workflow gain outweighs the penalty, but consistency becomes the default.

### Expected effects

1. High-importance duplicate shortcuts settle into stable positions across layers.
2. Workflow-specific shortcuts remain free to cluster where needed.
3. Users keep muscle memory for common actions across layer switches.
4. Layer differentiation is preserved without forcing relearning.
